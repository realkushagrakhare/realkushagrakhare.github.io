Real Engine 2.0

A 3D Game Engine built from scratch in progress using LWJGL and new OpenGL 3.0+ using the programmable pipeline. Some of the features developed:

  • VAOs, VBOs, Vertex and Fragment Shaders for optimization
  • OBJ and PLY file reader to load 3D models
  • Multiple point lights (with attenuation) and upto 4 directional lights as well as Specular Lighting
  • Terrain generation via heightmaps as well as procedural terrain using Perlin noise
  • Post Processing effects like lensflare, fog, cel shading, gaussian blur, contrast changer etc.
  • Rendering GUIs and distance field text rendering
  • Particle effects like bloom, fire with particle life cycle
  • Soft shadows using shadow mapping
  • Specular maps, bump maps and glow maps to specifiy object's shininess, normals and glow respectively
  • Created a OBJ and PLY file reader
  • Mouse picker to place objects in the scene
  • Textures with texture atlas and transparency
  • Skybox with Day-Night cycle

Project Water

Water simulation has a long history in computer graphics and has attracted hundreds of researchers. But the main reason it has fascinated me is the various phenomenon that occurs in a liquid like water like reflections, refractions, waves or even splashes. The major inspiration arose from Evan Wallaceâ€™s water rendering. There are various techniques to simulate water. I considered water as a planar mesh to triangles and distorting the mesh or the textures for various effects.

  • Simulation in modern OpenGL3.0+ with shaders in C++
  • Rendering waves by distorting the vertices of the mesh with a sine wave and then recalculating the normals in the vertex shader.
  • Storing the velocities of every vertex in a texture and using them to find out where the ripple has reached at every step, the vertices would lose velocity while the neighbouring ones would gain it, resulting in the ripple moving forward.
  • Real time ray tracer for refractions. Using the refracted rays calculated using camera, normal vector and light vector, calculated the intersection with the walls of the rays. This intersection was used to find out refraction texture at a particular point.
  • For caustics, found out where the light passing through a ripple would intersect the cubic box. At the point of intersection there could be a caustic, if the light fell at a particular angle.

Sociophobia VR

VR simulation for people with certain anxiety disorders in professional settings, such as speaking in front of a group at work or attending an interview. We see a large number of people with such issues and this project is to help them:

  • A scene with a small number of people in a typical team meeting or a presentation in a company
  • Server-client model, wherein majority of the controls, for controlling various events to create distraction like phone vibration, chair squeeking, people talking etc, are triggered from the server model, which controls the scene in the client
  • The patient (player) uses a HMD to view the scene (the audience) - and simultaneously talk through the presentation (considering it an actual presentation)
  • As the player starts presentation, the audience indulges in actions (animated) such as yawning or showing lack of interest, distract the speaker by turning and talking to another person etc. Other included events to create distraction are phone vibration, chair squeeking, person exiting the room. These actions would be controlled by a therapist.
  • Framework to add more such events using Factory Design Pattern.
  • Support to automatically trigger these events given the probability of each event.
  • Playback mode where the speaker can see his/her performance from a third viewpoint. Player's head movements are tracked while their talking is recorded, and these are used during playback.

This project was completed in collaboration with Shreyas Iyer.

3D Path Planning

The project was to built a path planning algorithm which creates paths for robots, self-driving cars, drones, etc considering various non-holonomic constraints. We solved the problem statement in the following steps:

  • Created the basic framework for path planning using RRT(Rapidly-exploring Random Trees).
  • Optimized the algorithm using cost function of A* to determine selection of nodes in the RRT.
  • Upgraded RRT into RRT* by constructing multiple short paths randomly organized as tree instead of one long path which yields asymptotically optimum solution.
  • Incorporated 2-Phase sampling to reach the goal faster.
  • Node pruning and Spline Fitting into the pipeline.
  • Combined Dubins and Reeds-Shepp Path Planning algorithms with RRT* for non-holonomic constraints.
